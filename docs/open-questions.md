# Offene Fragen

- Sollen Beträge wirklich negativ erlaubt sein, obwohl es `entry_type` gibt?
- Wie werden Buchungen mit mehreren Kategorien fachlich ausgewertet?
- Wie sollen Umbuchungen zwischen eigenen Konten modelliert werden?
- Welche Regel gilt später für CSV-Dubletten?
- Wie soll das Login später ausgestaltet werden?
- Welche Sicherheitsanforderungen gelten bei möglichem Public-Betrieb?

## Annahmen

- Der private Start reduziert zunächst den Umfang der Sicherheitsanforderungen, ersetzt aber keine spätere Sicherheitsplanung.
- CSV bleibt im MVP, auch wenn Mapping- und Importregeln erst später festgelegt werden.
